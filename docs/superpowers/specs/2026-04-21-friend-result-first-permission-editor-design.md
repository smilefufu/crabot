# 好友结果优先权限编辑设计

日期：2026-04-21

## 背景

当前 `对话对象管理` 中，`群聊` 和 `好友` 的权限编辑模型不一致：

- `群聊` 已经允许直接编辑当前生效权限
- `好友` 仍然只能选择 `permission_template`

这会让用户同时学习两套心智：

- 群聊编辑的是“结果”
- 好友编辑的是“来源”

对于管理者来说，这种差异没有必要，也会让好友权限编辑显得多余和绕。

## 目标

- 让 `好友` 权限编辑与 `群聊` 权限编辑保持一致
- UI 只展示和编辑“当前生效值”，不要求用户理解模板继承
- `好友` 成为私聊权限的直接管理主体
- `私聊池` 保持无归属、无权限的状态，不提供权限编辑

## 非目标

- 不改变 `私聊池` 的无权限定位
- 不改变 `master` 的权限模型
- 不删除后台已有的权限模板体系
- 不在这一轮里重构所有权限模板管理页

## 产品决定

### 1. 权限主体统一

权限主体固定为：

- `好友`：承接该好友统一私聊权限
- `群聊`：承接该群自己的权限
- `私聊池`：不承接权限

这意味着：

- 已归属私聊不再作为独立权限主体存在
- 非 `master` 好友不再只通过“权限模板”下拉框间接控制权限

### 2. 好友编辑模型统一为结果优先

好友权限编辑应与群聊一致：

- 布尔权限使用开关
- `storage` 直接编辑当前生效值
- `memory_scopes` 直接编辑当前生效值
- 不显示 `继承`
- 不显示 `重置为继承`
- 不显示 `覆盖存储权限`

用户看到什么，保存后就固定成什么。

### 3. 模板降级为后台实现细节

模板体系仍可保留在内部模型中，但不再是好友权限编辑的主入口。

用户侧不应该再被要求先理解：

- 这个好友绑定了哪个模板
- 模板里到底开了什么
- 改模板和改结果的区别

对于好友权限页面，模板来源应退出主交互。

## 对象行为

### 好友

好友详情中的权限区块改为：

1. 直接权限编辑入口
2. 当前生效权限摘要
3. 保存后的显式好友权限配置

保存行为采用完整显式配置，而不是增量 toggle。

### 群聊

群聊继续保持当前的结果优先模型。

好友与群聊的差别只保留在“对象类型”，不保留在“权限编辑方式”。

### 私聊池

私聊池条目继续只承担归属动作：

- `归到已有好友`
- `从私聊新建好友`

在归属前：

- 不展示权限编辑器
- 不提供权限摘要

## 界面设计

### 好友详情

当前好友详情中的“权限模板”选择区，需要替换为与群聊一致的权限编辑区。

展示原则：

- 主文案回答“现在是什么”
- 控件直接作用于生效值
- 不向用户暴露来源链路

### 编辑项

好友权限编辑区包含：

1. `tool_access`
2. `storage`
3. `memory_scopes`

表现形式与群聊保持一致：

- `tool_access`：逐项开关
- `storage`：启用开关 + 路径 + 访问级别
- `memory_scopes`：`当前会话 / 自定义范围`

### 需要移除的文案

好友权限区域中不应再出现：

- `权限模板`
- `继承`
- `重置为继承`
- `覆盖`
- 任何要求用户理解 `template_id` 的文案

## 数据行为

### 1. 好友权限作为独立读模型

好友需要有自己的统一私聊权限读模型。

读取好友详情时，前端应拿到：

- 好友当前显式配置
- 好友当前解析后的生效权限

### 2. 显式保存

保存好友权限时，应提交完整显式配置：

- 完整 `tool_access`
- 完整 `storage`
- 完整 `memory_scopes`

保存后，该好友的私聊权限以显式值为准。

### 3. 私聊归属关系

某个私聊一旦归属到好友：

- 权限读取走好友对象
- 不再要求管理员单独为该私聊 session 再做一份权限判断

## 迁移规则

已有非 `master` 好友如果当前只绑定了 `permission_template`，则前端首次进入好友权限编辑时，应展示模板解析后的实际结果。

用户第一次保存后：

- 该好友生成一份显式好友权限配置
- 后续不再依赖用户理解模板来源

## 风险

- 如果好友权限解析结果与实际私聊运行时权限不一致，UI 会误导用户
- 如果好友和群聊最终仍走两套后端规则，前端统一会变成表面一致
- 模板仍然保留时，需要明确其在后台中的角色，避免未来又重新回到“模板就是主入口”

## 验收标准

### 交互

- 非 `master` 好友不再只显示“权限模板”下拉框
- 好友权限编辑器与群聊权限编辑器使用同一种结果优先交互
- 私聊池不出现权限编辑入口

### 行为

- 打开好友权限编辑时，用户能直接看到当前生效值
- 修改并保存后，重新打开能看到完全一致的值
- 好友对象成为已归属私聊的统一权限主体

### 一致性

- 好友和群聊在权限编辑心智上保持一致
- 用户不需要再区分“这里改的是模板”还是“这里改的是结果”
